Not a Spotify clone

A streaming feed optimizes individual consumption. Dotify starts with a work, lets someone open a shared space around it, and keeps the artist in control.

For listeners

Open a shared link, see who welcomes you, and hear the same moment without an account or wallet checkpoint.

For artists

Publish through an artist-owned SmartRuntime and choose how each listening door opens, how value splits, and how the work travels.

For communities

Turn isolated listening into temporary cultural commons without building an advertising profile or an infinite social feed.

Production spine before spectacle

The first Thresholds slice joins product clarity to enforceable boundaries: room guests stay wallet-free, public room payloads omit protected source references, and the backend verifies the audio identity it encrypts. Artist publication remains paused while the deployed owner boundary is remediated.

01

Enter and hear

A room link resolves real host, work, and presence metadata before the guest chooses a local name and enters.

02

Protect the work

Room guests receive the ephemeral host stream, never a content key or protected source reference. Individual key release follows a server-side access decision.

03

Preserve sovereignty

The source now restricts registration to the runtime owner. New runtime creation and release registration stay paused until existing runtimes, the future factory/directory, and catalog continuity are all verified.

04

Prove critical flows

Contracts, API, signaling, unit, and deterministic browser tests cover the functional spine; real-device WebRTC and DAV2 validation remain release evidence.

Design principles

The link is the threshold

Show the host, work, real presence, and sound state first. Ask for identity or confirmation only when the chosen action requires it.

Fail closed on access

Ambiguous access decisions deny playback. No silent fallbacks. The policy is the truth.

Invisible complexity

The user sees a shared link. The infrastructure handles keys, chains, and access policy underneath.

Artist sovereignty

Artists publish through runtimes they own. No platform intermediary controls the terms of access.

Repository memory

Context lives in docs/backlog/ and docs/context/. AI agents must read AGENTS.md or CLAUDE.md before implementing.

The blockchain belongs in the guarantees. The music, the artist, and the people belong at the threshold.
