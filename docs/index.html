Not a Spotify clone

Spotify optimizes individual consumption. Dotify optimizes shared musical presence, artist sovereignty, and cultural transmission. The first felt value should be simple:

create a room → share a link → listen together

The product promise

For listeners

Discover music through people, not only algorithms. Join a room and enter the same listening moment.

For artists

Publish through artist-owned SmartRuntimes with access policy, royalties, and protected audio references.

For communities

Turn isolated listening into a living cultural commons where music travels through human presence.

Production priorities

1. Secure keys

Move Pinata uploads and content-key delivery behind a backend boundary.

2. Unlock access

Use wallet-signed key requests and on-chain access checks before full playback.

3. Stabilize rooms

Hosted signaling, join links, heartbeats, expiry, reconnect states, and clear errors.

4. Test critical flows

Classic unlock, artist publish, and room join must be deterministic and CI-covered.

Repository memory

The project keeps its execution memory in docs/backlog/ and its long-term product, technical, and philosophical memory in docs/context/. AI coding agents should read AGENTS.md or CLAUDE.md before implementing and keep this page aligned after relevant changes.

Dotify must be as simple as a shared link, as alive as a listening room, as fair as an auditable access policy, and as deep as a cultural commons.
