# Dotify context memory

This directory materializes the long-term memory of the Dotify/Muzinga project inside the repository.

It exists so human contributors and AI coding agents can preserve coherence across product, engineering, and philosophical decisions.

## Files

| File | Purpose |
| --- | --- |
| `dotify-product-memory.md` | Product vision, user roles, differentiation, competitor-informed positioning |
| `dotify-technical-memory.md` | Current architecture, production risks, engineering priorities |
| `dotify-philosophical-north-star.md` | Philosophical and social thread that must guide product details |
| `dotify-agent-working-protocol.md` | How AI agents should work inside this repository |

## Canonical execution references

- Root agent guidance: `AGENTS.md`
- Claude Code guidance: `CLAUDE.md`
- Production backlog: `docs/backlog/README.md`
- Project specification: `spec.md`
- Product overview: `README.md`
- Presentation deck: `Dotify_presentation.pptx` when available locally

## Rule

If a technical decision makes the product more complex for users without strengthening artist sovereignty, shared listening, or production security, challenge it.

Dotify must remain coherent in the details.
